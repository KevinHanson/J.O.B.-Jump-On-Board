Linux Commands

Introduction

The Linux operating system has been around for a number of decades and has roots going back to the dawn of the internet. Overtime many great tools have been developed that enable Linux computers to perform automated and complex procedures over and over again. There are many data tools that can ease the burden of data analysis and manipulation. Therefore, the majority of the systems we maintain at CTS-IT run Linux. Even the Mac laptops that we use have Linux underpinning in the OSX operating system. In this module you will learn about many essential Linux commands that enable us to do our jobs. The goal for this modulue is:

Increase employees' knowledge of Linux commands as it relates to the practice of data manipulation and sharing in health-related datasets.

This module is organized into six sections that focus on ever advancing concepts in Linux commands. Each section has an introduction and materials that lead to exercises for you to do online and at your own pace. At the conclusion of these exercises you will take an online assessment and finish your training in a face-to-face review with a team lead that will give you a chance to ask questions and cement your understanding of the commands you have just practiced.

File Commands

Objectives

Upon completion of this section, you will be able to:

  • Explain basic Linux commands and what they do
  • Understand when basic Linux commands are used

Description

This section introduces the basic commands used in Linux. These commands are rudementary in nature and allow someone to navigate their computer, create and move directories, look at running processes or elevate rights.

The basic commands that will be practiced are:

  • cd – Changes the current working directory in the command line console.
  • exit – Exits out of the current program, terminates the current command line terminal, or logs you out of a Unix network depending on the context.
  • ls – List all of the contents of a specified directory. If no directory is specified, it will use the current directory.
  • man – There’s a running gag in the Linux community that man is the only command you need to know. It stands for manual, and it will give you detailed information on commands and aspects of Linux.
  • pwd – Displays the current working directory for the command line terminal. Good for when you’ve lost track of where you are in your system.
  • sudo – Runs commands as root, which means no limitations due to permissions.
  • cat - Concatenate files and print on the standard output.

System Commands

Objectives

Upon completion of this section, you will be able to:

  • Explain Linux system information commands and what they do
  • Understand when Linux system information commands are used

Description

This section introduces the system information commands used in Linux. These commands are simple but can be used in powerful ways. These commands allow someone to display their current computer name, display the current date and time and show how much disk space is available on the computer.

The system information commands that will be practiced are:

  • date – Prints out the current system date and time. Specified parameters can change the format of the output.
  • df – Reports the disk space usage for the file system.
  • hostname – Displays the name of the current host system.
  • top – Displays all of the top processes in the system, by default sorted by CPU usage.
  • uptime – Reports how long the system has been running since last boot. Extremely useful for servers.

File Handling

Objectives

Upon completion of this section, you will be able to:

  • Explain Linux file handling commands and what they do
  • Understand when Linux file handling commands are used

Description

This section introduces the file handling commands in the Linux operating system. These commands will allow you to create, remove, move, copy and edit files on your computer from the command line. These tools are used get files into a place where you can begin to work and analyze them for further work.

The file handling commands that will be practiced are:

  • cp – Copies files to a new location with a new name depending on the parameters. Can copy directories too, whether recursively (includes all subdirectories) or not.
  • mkdir / rmdir – Creates a directory (mkdir) or deletes a specified directory (rmdir). Directories can only be created and deleted within directories that you have permission in.
  • mv – Moves files and directories to another location. Can be used to rename files and directories by keep their source and destination locations the same.
  • open – Opens a specified file using the default system application for files of its type.
  • rm – Remove and remove directory. Used to delete files and directories from the system, whether one at a time or in batch.
  • Touch - Create new empty files on the filesystem.

Network Commands

Objectives

Upon completion of this section, you will be able to:

  • Explain Linux network commands and what they do
  • Understand when Linux network commands are used

Description

This section introduces the network commands that are used to connect to servers, pull files from the internet, pull data from systems and copy files between servers. These commands are used often in our work and will be valuable for your success here.

The network commands that will be practiced are:

  • wget – downloads files from the Internet at the specified URL to your system.
  • rsync – a utility for efficiently transferring and synchronizing files across computer systems, by checking the timestamp and size of files.
  • scp – allows files to be copied to, from, or between different hosts.
  • curl – a tool to transfer data from or to a server, using one of the supported protocols (HTTP, HTTPS, FTP, FTPS, SCP, SFTP, TFTP, DICT, TELNET, LDAP or FILE). The command is designed to work without user interaction.
  • ssh - is a secure protocol for remote logins.
